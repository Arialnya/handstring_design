# ESP8266 心率监测与 RFID 系统

这是一个基于 ESP8266 的心率监测与 RFID 识别系统，集成了 Web 服务器功能，可以通过网页实时查看测量数据。

## 硬件要求

- ESP8266 NodeMCU 开发板
- MFRC522 RFID 模块
- MAX30105 心率传感器
- 面包板和连接线
- USB 数据线（用于编程和供电）

## 软件要求

### Arduino IDE 设置
1. 安装 ESP8266 开发板支持包
   - 打开 Arduino IDE
   - 进入 "文件" -> "首选项"
   - 在 "附加开发板管理器网址" 中添加：
     ```
     http://arduino.esp8266.com/stable/package_esp8266com_index.json
     ```
   - 进入 "工具" -> "开发板" -> "开发板管理器"
   - 搜索并安装 "ESP8266"

2. 安装必需的库
   - ESP8266WiFi
   - ESP8266WebServer
   - MFRC522
   - MAX30105
   - Wire
   - SPI

### 开发板设置
- 开发板：NodeMCU 1.0 (ESP-12E Module)
- 上传速度：115200
- CPU 频率：80 MHz
- Flash 大小：4MB (FS:2MB OTA:~1019KB)

## 接线说明

### RFID 模块 (MFRC522) 接线
| MFRC522 引脚 | ESP8266 引脚 | 说明 |
|-------------|-------------|------|
| RST         | D3 (GPIO0)  | 复位引脚，启动时需要保持高电平 |
| SS (SDA)    | D4 (GPIO2)  | 片选引脚，启动时需要保持高电平 |
| MOSI        | D7 (GPIO13) | SPI MOSI |
| MISO        | D6 (GPIO12) | SPI MISO |
| SCK         | D5 (GPIO14) | SPI SCK |
| VCC         | 3.3V        | 电源正极 |
| GND         | GND         | 电源负极 |

### 心率传感器 (MAX30105) 接线
| MAX30105 引脚 | ESP8266 引脚 | 说明 |
|--------------|-------------|------|
| SDA          | D2 (GPIO4)  | I2C 数据线 |
| SCL          | D1 (GPIO5)  | I2C 时钟线 |
| RESET        | D8 (GPIO15) | 复位引脚，启动时需要保持低电平 |
| VCC          | 3.3V        | 电源正极 |
| GND          | GND         | 电源负极 |

## 重要注意事项

1. 启动引脚状态要求：
   - GPIO0 (D3) 必须保持高电平
   - GPIO2 (D4) 必须保持高电平
   - GPIO15 (D8) 必须保持低电平
   - 代码中已通过 INPUT_PULLUP 和 OUTPUT 设置确保正确的启动状态

2. 电源供应：
   - 所有模块使用 3.3V 供电
   - 确保电源能够提供足够的电流（建议至少 500mA）
   - 如果使用外部电源，确保与 ESP8266 共地

3. 接线注意事项：
   - 所有连接必须牢固
   - 避免信号线与电源线平行布线
   - 如果遇到干扰，可以尝试缩短连接线长度
   - 建议使用面包板时，将数字和模拟部分分开布局

## 使用说明

1. 上传代码：
   - 打开 Arduino IDE
   - 选择正确的开发板和端口
   - 修改 WiFi 设置（ssid 和 password）
   - 上传代码

2. 首次使用：
   - 打开串口监视器（波特率：115200）
   - 等待设备连接到 WiFi
   - 记下显示的 IP 地址

3. 访问网页界面：
   - 在浏览器中输入设备 IP 地址
   - 等待页面加载完成
   - 按照提示进行操作：
     1. 刷卡开始测量
     2. 将手指放在心率传感器上
     3. 等待测量结果
     4. 移开手指结束测量

## 故障排除

1. 设备无法启动：
   - 检查启动引脚状态
   - 确认电源供应是否稳定
   - 检查接线是否正确

2. WiFi 连接失败：
   - 确认 WiFi 名称和密码正确
   - 检查 WiFi 信号强度
   - 尝试重启设备

3. 传感器无响应：
   - 检查接线是否正确
   - 确认电源电压是否稳定
   - 尝试重置传感器

4. 网页无法访问：
   - 确认设备已成功连接到 WiFi
   - 检查 IP 地址是否正确
   - 确保设备与访问设备在同一网络

## 技术支持

如果遇到问题，请：
1. 检查接线是否正确
2. 确认所有库都已正确安装
3. 查看串口监视器的输出信息
4. 如果问题持续，可以尝试重新上传代码
